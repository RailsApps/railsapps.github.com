h1. Ruby on Rails Version History h4. by Daniel Kehoe _Last updated 27 August 2015_ A history of Ruby on Rails versions with summaries and advice. h4. If You Are New to Rails If you're new to Rails, see "What is Ruby on Rails?":http://railsapps.github.io/what-is-ruby-rails.html, the book "Learn Ruby on Rails":http://learn-rails.com/learn-ruby-on-rails.html, and recommendations for a "Rails tutorial":https://tutorials.railsapps.org/rails-tutorial. !http://twitter-badges.s3.amazonaws.com/t_logo-a.png(Follow on Twitter)!:http://www.twitter.com/rails_apps Follow "@rails_apps":http://twitter.com/rails_apps on Twitter for updates and timely Rails tips. h4. From the RailsApps Project This is an article from the RailsApps project. The "RailsApps project":http://railsapps.github.io/ provides open source applications and detailed tutorials for Rails developers. For Rails installation instructions or upgrade advice, see the articles: * "Installing Rails":http://railsapps.github.io/installing-rails.html * "Updating Rails":http://railsapps.github.io/updating-rails.html h2. Stay Informed The "railsdiff.org":http://railsdiff.org/ site shows differences between versions of the Rails default application. These two weekly email newsletters will keep you informed about new Rails releases: * "Ruby Weekly":http://rubyweekly.com/ * "Green Ruby News":http://greenruby.org/ Another weekly email newsletter is more technical, and focused on code arriving in the next version of Rails: * "This Week in Rails":http://rails-weekly.goodbits.io/ You also can follow "@rails_apps":http://twitter.com/rails_apps on Twitter for news. h2. Rails 4.2 Rails 4.2 is the current stable version. Rails 4.2.4 is the newest release version. Notable additions in Rails 4.2 include "Active Job":https://github.com/rails/rails/tree/master/activejob for queueing tasks to run separately from the user request-response cycle. Learn more about Rails 4.2: * "Rails 4.2 Release Announcement":http://weblog.rubyonrails.org/2014/8/20/Rails-4-2-beta1/ * "Ruby on Rails 4.2 Release Notes":http://edgeguides.rubyonrails.org/4_2_release_notes.html * "Lesser-known Features in Rails 4.2":http://www.justinweiss.com/blog/2014/08/25/the-lesser-known-features-in-rails-4-dot-2/ _What You Need to Know:_ Rails 4.2 is the current stable version. Upgrade older projects to Rails 4.2 now. h4. Rails 4.2 Release History * Rails 4.2.4 was released August 24, 2015. See "Rails 4.1.0.beta1":http://weblog.rubyonrails.org/2013/12/17/Rails-4-1-beta1/. h2. Rails 4.0 Rails 4.0 was released June 25, 2013. Two release candidates were released beginning April 29, 2013. The "Rails 4.0: Beta 1 release":http://weblog.rubyonrails.org/2013/2/25/Rails-4-0-beta1/ announcement offers an overview of new features. You can learn more about Rails 4.0: * Ebook: Andy Lindeman "Upgrading to Rails 4":http://www.upgradingtorails4.com/ * Ebook: Philip De Smedt "Upgrade to Rails 4":http://upgradetorails4.com/ * Blog post: Mario Chavez "Upgrading a Rails application to 4.0":http://blog.crowdint.com/2013/03/07/upgrading-a-rails-application-to-4-0.html * Script: Michael de Silva "upgrade_rails_3.2.12_to_4.0.0.beta1":https://github.com/bsodmike/upgrade_rails_3.2.12_to_4.0.0.beta1 * Video: Code School "Rails 4: Zombie Outlaws":http://rails4.codeschool.com/videos * Article: RailsGuides "Upgrading from Rails 3.2 to Rails 4.0":http://edgeguides.rubyonrails.org/upgrading_ruby_on_rails.html#upgrading-from-rails-3-2-to-rails-4-0 (work-in-progress document) _What You Need to Know:_ The biggest changes are the use of _strong parameters_, which requires rewriting controller methods that process forms, and _Turbolinks_, which requires rewriting JavaScript code that depends on page loading. h4. Improvements over Rails 3.1 include faster execution in development mode. The speed of resolving routes also improved. There are many tiny (but helpful) improvements throughout the framework. Rails 3.2 deprecates plugins so you must find Ruby gems to replace any plugins used in earlier Rails projects. You can use Ruby 1.8.7 but you should use Ruby 1.9.3 or Ruby 2.0. See the "Rails 3.2.0 Announcement":http://weblog.rubyonrails.org/2012/1/20/rails-3-2-0-faster-dev-mode-routing-explain-queries-tagged-logger-store and "Ruby on Rails 3.2 Release Notes":http://guides.rubyonrails.org/3_2_release_notes.html (with upgrade instructions). There's a Railscast from Ryan Bates titiled "Upgrading to Rails 3.2":http://railscasts.com/episodes/265-rails-3-1-overview. Michael Hartl's popular Ruby on Rails Tutorial book covers Rails 3.2 ("Ruby on Rails Tutorial Version 3.2":http://ruby.railstutorial.org/ruby-on-rails-tutorial-book?version=3.2). Finally, José Valim highlights his "Five Favorite 'Hidden' features in Rails 3.2":http://blog.plataformatec.com.br/2012/01/my-five-favorite-hidden-features-in-rails-3-2/. _What You Need to Know:_ Rails 3.2.16 is far more secure than earlier versions. If an application uses an older version of Rails, you should update to Rails 3.2.16 immediately and then begin an upgrade to Rails 4.0. h4. See "Rails 3.2.10, 3.1.9, and 3.0.18 have been released":http://weblog.rubyonrails.org/2013/1/2/Rails-3-2-10--3-1-9--and-3-0-18-have-been-released/ * Rails 3.2.9 was released November 12, 2012 with many small bug fixes. See the "Rails 3.2.2 Announcement":http://weblog.rubyonrails.org/2012/3/1/ann-rails-3-2-2-has-been-released. * Rails 3.2.1 was released January 26, 2012 with some minor fixes. See "Ruby on Rails v3.2.1 Changelogs":https://gist.github.com/1685812. h2. Rails 3.1 Rails 3.1.0 was released August 31, 2011. Eight release candidates were released beginning May 21, 2011. Rails 3.1 introduced the "asset pipeline":http://www.youtube.com/watch?v=cGdCI2HhfAU which enables proper organization of CSS and JavaScript. Other features include "HTTP streaming":http://railscasts.com/episodes/266-http-streaming, default jQuery, reversible migrations, mountable engines, identity map, prepared statements, Rack::Cache on by default, forced SSL, role-based mass-assignment protection, has_secure_password, and custom serializers. See the "Rails 3.1 release candidate announcement":http://weblog.rubyonrails.org/2011/5/22/rails-3-1-release-candidate for details. There's a "Rails 3.1 Overview":http://railscasts.com/episodes/265-rails-3-1-overview from Ryan Bates and a "changelog":https://gist.github.com/958283 for all the details. Yehuda Katz explains "What’s Up With All These Changes in Rails?":http://yehudakatz.com/2011/06/14/what-the-hell-is-happening-to-rails/. The best overview of Rails 3.1 is Michael Hartl's "chapter on Rails 3.1":http://ruby.railstutorial.org/chapters/rails-3-1#top in his Ruby on Rails Tutorial book. _What You Need to Know:_ The greatest impact for developers moving from Rails 3.0 to 3.1 is the new location for CSS and JavaScript files and changes to the application layout file. See the official "Rails 3.1 Asset Pipeline guide":http://guides.rubyonrails.org/asset_pipeline.html for details. h2.
